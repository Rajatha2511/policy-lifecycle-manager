SECTION 3 — Demo /generate-report (1:30 - 3:00)
==================================================

WHAT TO SAY:
"The first endpoint I built is /generate-report. This takes insurance
policy text and streams an AI-generated summary back in real time
using Server-Sent Events. Watch how the response streams token
by token — this gives a much better user experience than waiting
for the full response."

COMMAND TO RUN (Terminal 2):
Invoke-RestMethod -Uri "http://127.0.0.1:5000/generate-report" -Method POST -ContentType "application/json" -Body '{"text": "This health insurance policy provides comprehensive coverage for hospitalization due to illness or accidental injury. The policy covers up to 10 lakhs per year including room charges, doctor fees, and medication costs. Pre-existing conditions are excluded for the first 3 years."}'

EXPECTED OUTPUT:
data: {"token": "This"}
data: {"token": " health"}
data: {"token": " insurance"}
... (streaming tokens)
data: {"done": true}

WHAT TO SAY AFTER:
"You can see each token arriving individually — this is real-time
streaming powered by Groq's LLaMA 3.3 70B model. The done:true
at the end signals the stream is complete."

SECTION 4 — Demo /analyse-document (3:00 - 4:30)
==================================================

WHAT TO SAY:
"The second endpoint is /analyse-document. This is more sophisticated
— it analyzes a policy document and returns structured JSON with
a summary, findings, severity levels, and direct quotes from the
document. I also added Redis caching here — the first call hits
the AI, but any repeat call with the same text returns instantly
from cache."

COMMAND TO RUN (Terminal 2):
Invoke-RestMethod -Uri "http://127.0.0.1:5000/analyse-document" -Method POST -ContentType "application/json" -Body '{"text": "This health insurance policy provides comprehensive coverage for hospitalization due to illness or accidental injury. The policy covers up to 10 lakhs per year including room charges, doctor fees, and medication costs. Pre-existing conditions are excluded for the first 3 years."}' | ConvertTo-Json -Depth 10

EXPECTED OUTPUT:
{
    "document_length": 278,
    "document_summary": "Comprehensive health insurance policy with exclusions.",
    "findings": [
        {
            "description": "The policy provides extensive coverage for various medical expenses.",
            "severity": "low",
            "source_text": "comprehensive coverage",
            "title": "Comprehensive Coverage",
            "type": "insight"
        },
        {
            "description": "Policyholders with pre-existing conditions may not be covered for the first 3 years.",
            "severity": "medium",
            "source_text": "pre-existing conditions",
            "title": "Pre-existing Conditions Excluded",
            "type": "risk"
        }
    ],
    "timestamp": "2026-05-07T12:26:10.974980+00:00"
}

WHAT TO SAY AFTER:
"Notice the structured output — we get a summary, findings with
types (insight or risk), severity levels from low to high, and
direct quotes from the document as source text. This is exactly
the kind of structured data the frontend needs to display
professional policy analysis."

SHOW REDIS CACHE (run same command again):
"Now watch what happens when I call the same endpoint again with
the same text..."

WHAT TO SAY AFTER SECOND CALL:
"It returns instantly — that's the Redis cache working. The result
was stored for 1 hour so we don't hit the AI API unnecessarily.
You can also see 'Cache hit' in the Flask terminal."

SECTION 5 — Demo /batch-process (4:30 - 5:30)
==================================================

WHAT TO SAY:
"The third endpoint is /batch-process. This accepts up to 20 policy
items in a single request and processes each one with a 100ms delay
to avoid rate limiting. It's perfect for bulk processing multiple
policies at once."

COMMAND TO RUN (Terminal 2):
Invoke-RestMethod -Uri "http://127.0.0.1:5000/batch-process" -Method POST -ContentType "application/json" -Body '{"items": ["Health insurance policy covering hospitalization up to 10 lakhs per year.", "Life insurance policy providing coverage of 1 crore for the insured.", "Vehicle insurance covering third party liability and own damage."]}' | ConvertTo-Json -Depth 10

EXPECTED OUTPUT:
{
    "results": [
        {
            "index": 0,
            "result": "This health insurance policy provides coverage for hospitalization expenses up to 10 lakhs per year.",
            "status": "success"
        },
        {
            "index": 1,
            "result": "This life insurance policy provides 1 crore coverage to the insured.",
            "status": "success"
        },
        {
            "index": 2,
            "result": "This vehicle insurance covers third party liability and own damage.",
            "status": "success"
        }
    ],
    "timestamp": "2026-05-07T12:28:23.067542+00:00",
    "total": 3
}

WHAT TO SAY AFTER:
"Each item is processed individually and returns its own index,
status, and result. If any item fails, only that item shows an
error — the rest still succeed. This makes it robust for
production use."
